# Claude Code 실행 프롬프트

아래 블록을 Claude Code 세션에 그대로 붙여넣으세요.

```text
이 저장소는 한국주식 단기스윙 자동 분류 시스템의 인수인계 패키지다.
아래 규칙을 반드시 지켜라.

[먼저 읽기]
1. 다음 문서를 모두 먼저 읽어라(순서대로): PROJECT_SPEC.md, ARCHITECTURE.md,
   DATA_MODEL.md, TASKS.md, ACCEPTANCE_TESTS.md, README_HANDOFF.md, OPEN_DECISIONS.md.
2. 그 다음 현재 프로젝트 상태를 먼저 검사하라: 디렉토리 구조, 기존 코드(app/, dashboard/,
   scripts/, tests/), data/watchlist.csv, 테스트 통과 여부(pytest -q)를 확인하고
   무엇이 이미 구현돼 있고 무엇이 비어 있는지 요약 보고하라.

[작업 방식]
3. 곧바로 전체 코드를 한꺼번에 만들지 마라. TASKS.md의 T1~T14를 순서대로 진행하라.
4. 각 단계마다 테스트를 작성/실행하라. 실패한 테스트를 먼저 수정한 뒤에만 다음 단계로 넘어가라.
5. 각 단계가 끝나면 의미 있는 단위로 Git 커밋을 분리하라(예: "T3: collector 폴백/재시도 구현").
6. 이미 통과하는 기존 테스트(20개)를 깨뜨리지 마라. 회귀가 나면 즉시 고쳐라.

[금지]
7. 가짜 금융 데이터를 만들지 마라. 외부 소스를 호출할 수 없으면 mock으로 테스트하되,
   합성 데이터는 "로직 검증 전용"임을 코드/주석에 명시하라. 실제 시세인 척 저장하지 마라.
8. 비밀키(API 키, DB 키, 봇 토큰, 비밀번호)를 코드에 적지 마라. 전부 환경변수/Secrets로 읽어라.
   .env는 커밋하지 마라. .env.example만 둬라.
9. data/watchlist.csv의 종목을 임의로 추가/삭제/변경하지 마라. 분류는 거래대금 계산 결과로만
   바뀐다. 워치리스트 원본 수정이 필요해 보이면 사용자 승인을 먼저 요청하라.

[분류 규칙 — 절대 준수]
10. 한국주식만 최근 6개월 일평균 거래대금으로 분류한다.
    6개월 일평균 = 실거래일 거래대금 합 ÷ 실거래일 수. 종가×평균거래량 추정 금지.
    1,000억 이하 → swing(정확히 1,000억 포함), 초과 → sector, 데이터 불완전 → hold.
    해외주식·M7은 분류 고정. 각 종목은 한 분류에만 속한다(중복 금지).
    수집 실패 시 이전 정상값을 0/빈값으로 덮어쓰지 마라.

[사용자 입력이 필요한 경우만 표시]
11. 사용자의 승인이나 입력이 반드시 필요한 경우(외부 계정/키 발급, OPEN_DECISIONS.md에서
    ' 사용자 결정 필요=예'인 항목, 워치리스트 변경)만 명확히 "[사용자 입력 필요]"로 표시하라.
    그 외에는 멈추지 말고 자율적으로 진행하라.

[시작]
먼저 1~2번(문서 읽기 + 상태 검사)을 수행하고, 발견한 현재 상태와 다음에 할 단계(T?)를
요약해 보고하라. 그 보고 후 T1부터 순차 구현을 시작하라.
```
