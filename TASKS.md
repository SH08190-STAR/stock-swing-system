# TASKS.md — 개발 작업 분해

각 작업: 목적 / 구현 / 선행조건 / 완료조건 / 테스트 / 위험요소.
Claude Code는 위에서 아래로 순차 진행하고, **각 단계 테스트 통과 후** 다음으로 넘어간다.
단계마다 Git 커밋을 분리한다.

---

## T1. 프로젝트 초기화
- **목적**: 빈 프로젝트 골격과 의존성 확정.
- **구현**: 디렉토리 구조, `requirements.txt`, `.gitignore`, `.env.example`, `config.py`(환경변수 로딩).
- **선행조건**: 없음.
- **완료조건**: `pip install -r requirements.txt` 성공, `python -c "import app.config"` 무오류.
- **테스트**: import 스모크 테스트.
- **위험**: 패키지 버전 충돌 → 버전 핀 최소화, 핵심만 고정.

## T2. 워치리스트 원본 입력
- **목적**: 확정된 종목 목록을 데이터로 적재.
- **구현**: `data/watchlist.csv` 로더(`app/watchlist.py`). 한국/해외/M7 분리 함수.
- **선행조건**: T1.
- **완료조건**: 한국 89 / 글로벌 79 / M7 8 로드, symbol 중복 0.
- **테스트**: `tests/test_watchlist.py` — 건수·중복·시장값(KR은 KOSPI/KOSDAQ) 검증.
- **위험**: **워치리스트 임의 변경 금지.** CSV가 단일 출처. 코드에서 종목 추가/삭제 금지.

## T3. 데이터 수집기
- **목적**: 한국 종목 6개월 일봉·거래대금 자동 수집.
- **구현**: `app/collector.py` — 거래일 판정, pykrx 주/FDR 예비, 재시도, 실패 시 hold.
- **선행조건**: T2.
- **완료조건**: 종목 1개 실수집 성공(네트워크 환경). 실패해도 예외 없이 hold 반환.
- **테스트**: `tests/test_collector.py` — 폴백·재시도·hold 경로(소스는 mock/monkeypatch).
- **위험**: 소스 차단/레이트리밋 → 폴백·간격(REQUEST_SLEEP)·재시도. 가짜 데이터 생성 금지.

## T4. 거래대금 계산기
- **목적**: 6개월 일평균 거래대금 정확 계산.
- **구현**: `app/classifier.py::_avg_6m_value` — 6개월 전~기준일 실거래일, 0/결측 제외, 합÷일수. 20일 평균·당일값도 산출.
- **선행조건**: T3.
- **완료조건**: 합성 일봉으로 계산값 일치.
- **테스트**: `tests/test_classifier.py` — 합계/일수/0원 제외 검증.
- **위험**: 종가×평균거래량 추정 **금지**. 거래정지 0원 포함 시 평균 왜곡 → 0 제외.

## T5. 분류 엔진
- **목적**: 1,000억 기준 swing/sector/hold 판정 + 변경 산출.
- **구현**: `classify_one/all`, `diff_classifications`.
- **선행조건**: T4.
- **완료조건**: 경계값(미만/정확히 1,000억/초과/거래일 부족/수집실패) 전 케이스 정확.
- **테스트**: `ACCEPTANCE_TESTS.md` 분류 항목 일체.
- **위험**: 경계 부등호 오류(이하 vs 미만) → 1,000억은 swing. 해외·M7 미적용 보장.

## T6. 데이터베이스
- **목적**: 영구 저장·중복 방지.
- **구현**: `app/database.py`, `scripts/schema.sql`, `scripts/initialize_db.py`.
- **선행조건**: T5, Supabase 프로젝트.
- **완료조건**: 테이블 생성, prices (code,date) upsert로 중복 미발생, hold 시 이전값 보존.
- **테스트**: `tests/test_database.py` — upsert 멱등성, None 필드 보존(가능하면 mock client).
- **위험**: 실패 시 0/빈값 덮어쓰기 → save_classification에서 None 필드 제외.

## T7. 변경 이력
- **목적**: 편입·복귀·보류 기록.
- **구현**: `database.record_history` + `runs` 기록.
- **선행조건**: T6.
- **완료조건**: 직전과 분류가 다른 종목만 history insert, 동일 분류 재기록 안 함.
- **테스트**: diff→insert 후 중복 재실행 시 추가 안 됨.
- **위험**: 같은 날 중복 기록 → (code,change_date,to_class) 가드.

## T8. 웹 대시보드
- **목적**: 9개 메뉴 조회 UI.
- **구현**: `dashboard/app.py` — 전체/스윙/섹터별/신규편입/복귀/거래대금순위/이력/보류/최신화시각 + 검색·필터·정렬·CSV·모바일·비밀번호.
- **선행조건**: T6.
- **완료조건**: 9개 메뉴 표시, 필터·검색·다운로드 동작.
- **테스트**: 로컬 `streamlit run`, 빈 DB/더미 DB 양쪽.
- **위험**: 대용량 렌더 → 캐시(ttl). 모바일 폭 → layout/컬럼 조정.

## T9. 텔레그램 알림
- **목적**: 일일/오류 자동 발송.
- **구현**: `app/notifier.py` — send 분기, 빌더 3종.
- **선행조건**: T5(메시지 데이터), 봇 토큰.
- **완료조건**: 일일요약·변경없음·오류 메시지 발송 확인.
- **테스트**: `tests/test_notifier.py` — 메시지 빌드 문자열, 전송은 mock.
- **위험**: 전송 실패가 파이프라인 중단 → False 반환, 예외 전파 금지.

## T10. 자동 실행
- **목적**: 무인 정기 실행.
- **구현**: `scripts/run_daily_update.py`, `.github/workflows/daily-update.yml`(cron+dispatch).
- **선행조건**: T6·T9.
- **완료조건**: 수동 실행으로 전체 파이프라인 1회 성공, 휴장일 스킵 확인.
- **테스트**: workflow_dispatch 실행, 휴장일 mock.
- **위험**: cron 시간대(UTC) 혼동 → 07:40 UTC. 휴장일 처리 누락 → 거래일 판정 필수.

## T12. 테스트

- **구현**: pytest 일체, `.github/workflows/tests.yml`(푸시 시 실행).

- **위험**: 네트워크 의존 테스트 불안정 → 외부 소스는 전부 mock.
