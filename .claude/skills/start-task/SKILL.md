# /start-task — 작업 시작 절차

새 작업을 시작할 때 아래 순서를 정확히 따른다.

## 절차

1. CLAUDE.md의 고정 규칙을 확인한다 (이미 로드되어 있으면 재확인만).
2. `docs/PROJECT_STATE.md`를 읽는다.
3. `docs/CURRENT_TASK.md`를 읽는다.
4. `docs/DECISIONS.md`에서 **현재 작업과 관련된 항목만** 읽는다.
   전체를 요약하거나 재출력하지 않는다.
5. `git status`를 실행해 작업 트리 상태를 확인한다.

## 충돌 검사

- CURRENT_TASK의 범위·상태와 실제 작업 트리(git status)가 충돌하면
  **구현하지 않고** 충돌 내용을 보고하고 멈춘다.
  - 예: CURRENT_TASK가 `대기`인데 로컬 변경이 존재
  - 예: 수정 허용 파일 밖의 파일이 이미 변경됨

## 컨텍스트 규칙

- CURRENT_TASK의 "읽을 파일" 목록에 있는 파일만 읽는다.
- 그 외 파일은 필요성이 확인된 경우에만 추가로 읽는다.
- 전체 저장소 탐색(무차별 Glob/Grep 스캔) 금지.

## 산출물

구현에 들어가기 전, 작업 범위를 **5줄 이내**로 요약해 제시한다:
목표 / 수정할 파일 / 완료 조건 / DB write 여부 / push 여부.
