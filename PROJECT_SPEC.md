# PROJECT_SPEC.md — 한국주식 단기스윙 자동 분류 시스템

> 이 문서는 확정된 요구사항의 단일 기준(source of truth)이다.
> Claude Code는 구현 중 충돌이 생기면 이 문서를 우선한다.

## 1. 프로젝트 배경

기존에 한국·글로벌 주식을 11개 산업 섹터로 묶은 통합 워치리스트(Z PICK)를 운영 중이다.
이 워치리스트에는 거래대금이 큰 대형주와, 단기 매매용으로 보는 거래대금이 작은 종목이 섞여 있다.
사용자는 **한국주식 중 거래대금이 작은 단기매매용 종목을 산업 섹터에서 분리해 '단기스윙종목'으로 따로 관리**하고자 한다.
이 분리를 매일 수작업으로 하는 것은 불가능하므로, 데이터를 자동 수집해 매 거래일 자동 분류하는 시스템을 만든다.

## 2. 목표

- 한국주식의 최근 6개월 일평균 거래대금을 외부 소스에서 자동 수집·계산한다.
- 1,000억 원 기준으로 단기스윙/기존섹터를 매일 자동 분류한다.
- 결과를 모바일·PC 웹 대시보드와 텔레그램으로 전달한다.
- 완성 후에는 Claude 실행 없이 독립적으로 자동 운영된다.

## 3. 범위 (In Scope)

- 한국주식(KOSPI/KOSDAQ) 일봉·거래대금 자동 수집
- 6개월 일평균 거래대금 계산(실거래일 기반)
- 단기스윙 분류 및 변경 이력 기록
- 웹 대시보드(조회·검색·필터·정렬·다운로드)
- 텔레그램 알림(교체 가능 구조)
- 매 거래일 자동 실행(휴장일 처리 포함)
- 인증/접근 제한, 비밀키 환경변수 관리

## 4. 확정 요구사항 (사용자 승인 완료 — 재질문 금지)

데이터·수집
- 사용자가 주가·거래대금을 직접 입력하지 않는다. 전부 외부 소스 자동 수집.
- 최근 6개월 **실제 거래일의 일별 거래대금**을 사용한다.
- 6개월 일평균 = **거래일 거래대금 합계 ÷ 실제 거래일 수**.
- **최근 종가 × 평균 거래량 방식의 임의 추정 금지.**
- 데이터 소스: KRX 공식 → 공공 API → 금융 API → pykrx → FinanceDataReader → 기타. 주/예비 분리.
- 실제 확인 불가한 데이터는 만들어내지 않는다.

분류
- 6개월 일평균 거래대금 **1,000억 원 이하 → 단기스윙**(정확히 1,000억도 단기스윙).
- **1,000억 원 초과 → 기존 산업 섹터 유지.**
- 해외주식·M7은 거래대금 기준 미적용, 분류 고정.
- 각 종목은 **기존 섹터 / 단기스윙 중 정확히 하나**에만 속한다(중복 금지).
- 데이터 불완전 종목은 임의 분류하지 않고 **확인 보류(hold)**.

운영
- 매 거래일 한국 증시 마감 후 자동 갱신(권장 16:40 KST). 휴장일·공휴일 처리.
- 스마트폰·PC 모두, 외부에서 브라우저로 접속 가능.
- 개인용 — 로그인/인증/비공개 링크/토큰 중 하나로 접근 제한.
- 매 갱신 결과를 메신저로 자동 발송. **변경이 없어도** 완료·오류 여부 발송.
- 메신저는 텔레그램 우선, 교체 가능 구조.
- 신규 편입·섹터 복귀 이력 저장.
- **수집 실패 시 이전 정상 데이터를 0/빈값으로 덮어쓰지 않는다.**
- 초기·유지 비용 무료 또는 저비용 우선.
- 완성 후 Claude 없이 독립 자동 운영.

보안
- API 키·DB 비밀번호·봇 토큰을 코드에 직접 쓰지 않는다.
- 환경변수 또는 플랫폼 비밀키 기능 사용. `.env`는 커밋 금지, `.env.example`만 제공.

## 5. 제외 범위 (Out of Scope)

- 해외주식의 거래대금 기준 재분류(고정 유지).
- 실시간 호가·분봉·체결 단위 데이터.
- 자동 매매·주문 실행(분석·분류·알림까지만).
- 종목 추천/투자 의견 생성.
- 워치리스트 종목의 임의 추가·삭제(거래대금에 따른 분류 이동만 허용).

## 6. 분류 기준 (정밀 정의)

```
avg_6m = Σ(실거래일 거래대금) / (유효 거래일 수)
  · 거래대금이 0 또는 결측인 날(거래정지 등)은 분모·분자에서 제외
  · 유효 거래일이 비정상적으로 적으면(기준 미만) hold 처리

if 수집 실패 또는 유효 거래일 부족:  classification = "hold"
elif avg_6m <= 100,000,000,000:      classification = "swing"   # 1,000억 이하
else:                                classification = "sector"  # 1,000억 초과

해외/M7: classification 고정(거래대금 미적용)
```

## 7. 자동화 기준

- 트리거: GitHub Actions cron, 평일 07:40 UTC(=16:40 KST). 수동 실행 버튼 제공.
- 실행 순서: 거래일 판정 → 수집 → 계산 → 분류 → 이전과 비교 → 저장 → 이력 → 메타 갱신 → 알림.
- 휴장일이면 수집을 스스로 스킵(오류 아님).
- 단계별 오류 격리: 한 종목·한 단계 실패가 전체 파이프라인을 멈추지 않는다.

## 8. 보안 기준

- 비밀값은 전부 환경변수/Secrets. 저장소·로그·메시지에 노출 금지.
- 대시보드는 공개 검색에 노출되지 않도록 인증(앱 비밀번호 + 플랫폼 viewer allow-list).
- 저장소는 비공개(private).
